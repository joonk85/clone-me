- **플랫폼 구독 UI**: 공개 `/pricing`(Mock·localStorage), **`/settings`** 구독 섹션, **앱 상단 헤더**(멤버 모드) 플랜·업그레이드 CTA (실결제는 Phase 3 `platform_subscriptions`)
- 인증 (이메일+비밀번호)
- 클론 만들기 + 자료 업로드 (카테고리 A~D 가이드 포함)
- RAG 기반 채팅 + 출처 표시
- 마이페이지 (멤버탭 + 마스터탭)
- 마스터 스튜디오 IA: **좌측 레일(섹션 + 내 클론) / 우측 작업영역** 구조
- 토큰 Mock 충전
- 클론 테스트 10가지 시나리오
- 보너스 토큰 만료 CRON
- 마스터 slug 자동 생성 (clone.me/@{slug} 공유 링크)
- 클론 컬러 피커 (Step 2 — 아바타/카드/채팅에 반영)
- v28 하드코딩 → Supabase DB 연동 (마켓/채팅/프로필/대시보드)

## 핵심 비즈니스 원칙
- 마스터가 없으면 플랫폼이 없다
- 초기엔 마스터 수익 최우선 보장
- 클론 품질이 플랫폼 신뢰의 핵심

## 클론 품질 우선순위
1. 어투/스타일 (SRT 자막 = 가장 임팩트 큼)
2. 핵심 지식 (PDF/블로그)
3. 케이스 & 경험
4. 고정 답변

## 기능 평가 프레임워크
```
1. 마스터가 없으면 안 되는가? → 최우선
2. 멤버 전환에 직접 영향? → 높은 우선순위
3. 나중에 추가 가능? → 후순위
4. 어드민 필요? → Phase 4
```
